# 油菜生长决策系统 - 需求共识文档

## 需求描述

基于JeecgBoot 3.8.2企业级AI低代码平台，开发一套油菜生长决策系统，实现多个种植基地的油菜生长全生命周期智能化管理和决策支持。系统采用基于规则的专家系统提供决策建议，通过外部数据接口获取大部分环境数据，支持云端部署和Web端访问。

## 验收标准

1. **多基地管理**：支持多个种植基地的数据隔离和统一管理
2. **数据采集**：支持外部数据接口对接和手动数据录入
3. **生长监测**：实现油菜生长阶段识别和状态跟踪
4. **决策支持**：基于规则引擎提供种植、施肥、病虫害防治等建议
5. **数据可视化**：提供生长数据和决策建议的可视化展示
6. **云端部署**：系统支持云端部署，可通过Web浏览器访问

## 技术实现方案

### 系统架构
- 基于JeecgBoot 3.8.2微服务架构
- 采用前后端分离设计
- 使用JeecgBoot多租户功能实现基地数据隔离
- 集成外部数据接口适配器

### 技术栈
- 后端：Spring Boot 2.7.18 + MyBatis-Plus + Shiro
- 前端：Vue3 + TypeScript + Vite6 + Ant Design Vue4
- 数据库：MySQL 5.7+
- 规则引擎：JeecgBoot内置规则引擎

### 核心模块
1. **基地管理模块**：管理多个种植基地信息和权限
2. **数据采集模块**：外部数据接口对接和手动数据录入
3. **生长监测模块**：油菜生长阶段识别和状态跟踪
4. **规则引擎模块**：决策规则管理和推理执行
5. **决策支持模块**：种植建议生成和展示
6. **可视化模块**：数据图表和决策建议可视化

## 技术约束

1. 必须基于JeecgBoot 3.8.2平台开发
2. 使用平台现有技术栈和组件
3. 遵循平台代码规范和架构模式
4. 利用平台现有功能（权限管理、代码生成器等）
5. 支持云端部署和微服务架构

## 集成方案

1. **与JeecgBoot平台集成**
   - 使用平台统一的用户认证和权限管理
   - 利用平台的低代码能力快速构建数据录入界面
   - 集成平台的报表和大屏工具展示数据

2. **外部数据接口集成**
   - 开发标准化的数据接口适配器
   - 支持气象数据、土壤数据、传感器数据等多种数据源
   - 实现数据清洗、转换和存储

3. **多基地数据隔离**
   - 利用JeecgBoot的多租户功能
   - 实现基地级别的数据隔离和权限控制
   - 支持跨基地数据汇总和分析

## 任务边界限制

1. **包含范围**
   - 油菜生长决策核心功能开发
   - 外部数据接口适配器开发
   - 多基地管理和数据隔离
   - Web端界面开发和数据可视化
   - 云端部署配置

2. **不包含范围**
   - 硬件传感器设备开发
   - 移动端APP开发
   - 第三方数据接口的商务合作
   - 复杂机器学习模型开发

## 验收标准

1. **功能验收**
   - 多基地管理功能正常
   - 数据采集接口稳定运行
   - 生长监测准确识别油菜生长阶段
   - 决策建议符合农业专家知识
   - 数据可视化清晰直观

2. **性能验收**
   - 系统响应时间小于3秒
   - 支持100个以上基地并发访问
   - 数据采集延迟不超过5分钟

3. **安全验收**
   - 基地间数据完全隔离
   - 用户权限控制有效
   - 数据传输加密

4. **部署验收**
   - 成功部署到云端环境
   - 系统稳定运行7×24小时
   - 支持浏览器跨平台访问

## 风险评估

1. **技术风险**
   - 外部数据接口稳定性风险
   - 规则引擎复杂度管理风险

2. **业务风险**
   - 农业专家知识获取和规则制定风险
   - 多基地需求差异风险

3. **缓解措施**
   - 设计灵活的数据接口适配器
   - 采用模块化规则管理
   - 与农业专家合作制定规则
   - 提供规则自定义功能
